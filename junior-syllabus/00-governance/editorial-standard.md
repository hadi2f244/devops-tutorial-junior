# Editorial Standard

## Required Session Sections

Each session document must include the following sections, in order:

1.  **Session Header**: `Session XX - Week X`
2.  **Topic**: A concise title for the session's content.
3.  **Session Summary**: A brief paragraph outlining the session's purpose and key takeaways.
4.  **Technical Topics**: A bulleted list of the core concepts and technologies covered.
5.  **Key Commands**: A table of essential commands introduced in the session.
6.  **Practical Scenario**: A description of the hands-on lab or activity.
7.  **LPIC-like Mapping**: Alignment with relevant Linux Professional Institute Certification objectives.
8.  **References**: Links to official documentation, practical tutorials, and other resources.
9.  **Assignment and Rubric**: Detailed instructions for the homework assignment and the grading criteria.
10. **Suggested Supplementary Content**: Optional materials for further learning.
11. **Detailed Learning Objectives**: A clear list of what students should be able to do after the session.
12. **Detailed Agenda**: A minute-by-minute breakdown of the session timeline (total 180 minutes).
13. **Step-by-step Teaching Path**: A guide for the instructor to deliver the content.
14. **Common Errors to Cover**: A section highlighting potential pitfalls and how to address them.
15. **Expanded Web References**: A categorized list of web links for deeper exploration.

## Source Quality Policy

For each session, references must include:

- **Official Docs (mandatory)**: Links to the official documentation for all tools and technologies.
- **Practical Technical Blogs/Tutorials (mandatory)**: High-quality, hands-on guides that demonstrate real-world application.
- **Certification Alignment**: Content should align with the concepts and syllabus outlined in `related-certificates.md`. Web searches should be conducted to ensure relevance and accuracy.
